# C/C++

直接用预处理器、编译器、汇编器、连接器处理成可执行文件。

- 最开始写一些简单的命令行程序

- 曾经试过学习开发win32应用，但很快就放弃了

应该是最早接触到的编程语言。学基础看的是《C++ Premiere Plus》。一开始用Visual Studio运行，后来改为VS Code+MinGW。

Linux下需要安装编译工具链gcc或g++，Windows上用MinGW代替。makefile可以自定义编译依赖和流程以减轻工作量。 Cmake可以自动生成makefile文件（以及其它支持的构建系统，如生成Visual Studio项目）。

# Java

面向对象编程语言，先编译成优化过的**字节码**，然后通过**Java虚拟机（JVM）**运行。最常用的虚拟机时HotSpot，可以在解释执行字节码时识别出**热点代码**并通过**即时编译（JIT）**技术将其优化呈本地机器码。语法和C++比较像。

一般用Intellij IDEA编写（虽然比较占内存）。不过我并没有写过很多Java。

- 唯一一次是在跟着B站上一个前后端结合项目做时，后端要用到Java Spring。那是一个构建Java EE应用的框架。但好像处理用IDEA自带的初始化工具之外，其它构建Java Spring应用的方式都较为繁琐（而且IDEA这个功能还要收费）。
顺带一提，会Java还能开发MC Java版模组。

只是运行的话需要JRE，开发需要JDK。

曾经用turtle绘图库写过一个展示多抛物线时间轨迹的程序

大一开发个人博客后端用Django框架时写过
做大模型研究时用得到
只需要按照Python解释器就可以。包管理需要pip。可以通过conda/miniconda来管理多个python环境，每个拥有自己的python版本、pip版本和第三方库。不过需要注意，为了防止在一个环境里调用到系统的pip而将库安装到系统里，有时需要python -m pip以调用环境中的pip。可以用which pip来区分。

# 汇编

逐条翻译成机器码后就能被CPU直接执行。

在Linux或macOS上可以通过objdump -d来查看可执行文件对应的汇编。gcc -S会让编译流程停在汇编器前，输出汇编代码文件。

# 多语言协作

[【中配】为什么有些项目使用多种编程语言 – Core Dumped](https://www.bilibili.com/video/BV1x9itB2Ei3)

C/C++、Rust、Fortran等语言可以先用各自的工具链处理成目标文件（或静态/动态库）后一起通过链接器。不过要注意不同语言对目标代码的约定不同（ABI），避免发生如寄存器不一致等错误。

对于解释型语言，由于解释器也是程序，而解释器内部使用专门的结构来表示解释语言的变量和函数；因此将这些结构暴露出去，并用编译语言操作它们，实现解释器的扩展。

有一些软件（如Godot）将性能要求高的操作交给编译语言，并且内置解释器，然后就能运行解释语言（GDScript）。
